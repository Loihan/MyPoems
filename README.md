# 🏮 MyPoems：个人数字化诗词书斋

**MyPoems** 是一款专为文学创作者与诗词爱好者打造的“数字化书斋”。它跳出了传统笔记软件的框架，将 **Node.js 的工程化能力** 与 **中国传统审美** 深度融合，旨在构建一个既能沉浸式阅读，又能进行深度文本分析的私密创作空间。

> **“笔落惊风雨，诗成泣鬼神。”** —— 在这里，每一行墨迹都有迹可循，每一个意象都被精准捕捉。

---

## 🏛️ 书斋雅色：核心特性

### 1. 沉浸式三栏架构
*   **持久化阅读面板**：首创“左导航-中列表-右阅读”的持久化布局。切换功能页面时，右侧阅读详情永不消失，实现真正的多任务并行。
*   **水墨双色调**：深度适配系统级深色模式。日间如素笺宣纸，清晰爽利；夜间如苍墨入潭，静谧护眼。

### 2. 意象簿 (Imagery Intelligence)
*   **意象挖掘**：系统自动提取作品中的高频意象词汇，支持一字/二字词频统计。
*   **楷书田字格**：选中意象即以传统楷体田字格呈现，重拾汉字结构之美。
*   **流变可视化**：通过双折线图分析特定意象在不同年份的“出现频次”与“年度创作占比”，洞察你的创作母题演变。

### 3. 笔耕录 (Writing Insights)
*   **创作热力图**：复刻 GitHub 风格的创作贡献矩阵，记录每一个挥毫泼墨的时刻。
*   **文学洞察**：
    *   **沉默期分析**：自动锁定最长的创作断层，提醒你重拾笔杆。
    *   **时节偏好**：基于二十四节气与月份雅称（孟春、仲夏等），分析你的灵感与季节的共鸣。

### 4. 编年史 (The Chronicle)
*   **中轴流向图**：以时间为轴，串联起生命中的每一篇华章。
*   **垂直刻度导航**：右侧悬浮年份快捷索引，支持一键溯流而上，瞬时定位往昔岁月。

### 5. 索引工程 (Tag Management)
*   **集合筛选**：支持多标签“交集”过滤，精准定位特定主题。
*   **批量清洗**：提供强大的后端 API 支持，实现标签的批量重命名、合并、删除及一键挂载。

---

## 🛠️ 技术匠心

*   **容器架构**: 基于 **Electron 25** 构建的桌面原生应用，彻底告别命令行弹窗，点击即入。
*   **核心逻辑**: 原生 JavaScript (ES6+) 实现 SPA（单页应用）路由，零框架负担，运行如飞。
*   **后端驱动**: Node.js + Express 搭建轻量化本地服务器。
*   **数据哲学**: 采用 **去数据库化** 设计。所有作品均以原生 `.json` 格式存储于 `/poems` 目录。
    *   *无感迁移*：拷贝文件夹即完成数据迁移。
    *   *透明读写*：每一篇诗词都是独立文件，不锁定数据，方便二次开发。

## 🚀 快速启程

### 下载运行
如果你是 Windows 用户，可直接在 **Releases** 页面下载最新版的 `.exe` 文件，与 `poems` 文件夹放在同一目录下即可使用。

### 源码构建
若需从源码进行个性化修改：

1.  **环境检查**: 确保安装了 Node.js (v16.0.0+)。
2.  **克隆项目**:
    ```bash
    git clone https://github.com/Loihan/MyPoems.git
    cd MyPoems
    ```
3.  **安装依赖**:
    ```bash
    npm install
    ```
4.  **调试启动**:
    ```bash
    npm start
    ```
5.  **构建桌面版 (.exe)**:
    ```bash
    npm run build
    ```

## 📂 馆藏结构

```text
MyPoems/
├── dist/            # 打包生成的桌面应用程序
├── html/            # 动态加载的功能模态片段
├── js/              # 系统逻辑中枢 (main 路由与模块化脚本)
├── css/             # 样式分块 (变量、布局、组件、页面)
├── poems/           # 你的私密诗词库 (.json 文件)
├── server.js        # 后端数据接口服务
├── electron-main.js # 桌面应用主进程配置
└── index.html       # 应用唯一入口
```

## 🔒 隐私与备份

*   **隐私保护**：项目 `.gitignore` 默认忽略 `poems/` 下的所有作品文件，确保你的私人创作不会被上传至公共代码仓库。
*   **数据安全**：请定期备份 `poems/` 文件夹。由于采用 JSON 存储，你可以轻松将其同步至坚果云、OneDrive 等同步盘。
