# 🏮 MyPoems - 个人数字化诗词书斋

MyPoems 是一款专为文学创作者打造的本地化诗词管理、赏析与深度分析系统。它不仅是一个存储文字的仓库，更是一个能够洞察创作者心境、打磨意象、沉浸阅读的私密数字化空间。

## ✨ 核心特性

### 🏛️ 极致的交互布局
- **三栏持久化布局**：创新的"导航-列表-详情"架构。右侧阅读面板持久存在，切换页面不丢失进度
- **水墨双色主题**：完美适配深色模式，白日如宣纸，深夜如墨韵
- **响应式排版**：根据诗词字数、类型（诗/词）智能调整对齐方式

### 📊 笔耕录 (深度统计)
- **创作热力图**：直观展示过去一年的创作频率，见证每一天的坚持
- **创作洞察**：
  - **最长的沉默**：记录那段没有文字掠过纸面的时光
  - **最密集的爆发**：捕捉灵感如泉涌的黄金月份
  - **偏爱的时节**：基于月份雅称（如孟春、仲夏）分析你的创作季节偏好

### ⏳ 编年史 (生命线)
- **中轴时间轴**：以流动的线条串联起每一篇作品
- **年份快捷跳转**：右侧悬浮刻度尺，一键溯流而上，寻找多年前的笔触

### 🖼️ 意象簿 (文本挖掘)
- **高频词云分析**：自动分析作品中 Top 50 的一字/二字意象
- **田字格预览**：以传统楷体展示选中的意象文字，极具书法美感
- **趋势可视化**：通过 Chart.js 绘制意象在各年份的"出现频次"与"年度占比"双折线图

### 🔖 索引管理 (工程化整理)
- **标签清洗**：支持标签的批量重命名、自动合并与删除
- **多维筛选**：支持多标签"交集"查询，轻松定位"无标签"作品
- **批量操作**：支持对筛选出的结果进行快捷的批量标签挂载

## 🛠️ 技术栈

- **后端**: Node.js + Express (轻量级 RESTful API)
- **前端**: 原生 JavaScript (ES6+) + CSS 变量 + CSS 多列布局 (瀑布流)
- **图表**: Chart.js (响应式数据可视化)
- **存储**: 本地 JSON 文件系统（无需数据库，零成本迁移备份）

## 📂 项目结构

```text
MyPoems/
├── css/             # 样式分块 (variables, base, layout, components, pages)
├── html/            # 页面功能片段 (view, add, manage, tags, timeline...)
├── js/              # 逻辑控制中心 (main 路由及各模块脚本)
├── poems/           # 诗词数据目录 (JSON 格式，由 .gitignore 保护隐私)
├── index.html       # 应用唯一入口
├── server.js        # 后端 Express 服务器
└── package.json     # 项目依赖配置
```

## 🚀 快速开始

### 1. 环境准备
确保已安装 Node.js (推荐 LTS 版本)

### 2. 克隆与安装
```bash
git clone https://github.com/Loihan/MyPoems.git
cd MyPoems
npm install
```

### 3. 运行
```bash
npm start
```
启动后访问 http://localhost:3000
## 🔒 隐私与备份

**隐私保护**：本项目默认在 `.gitignore` 中忽略了 `poems/*.json`。这意味着你的私人作品不会被同步到公共的 GitHub 仓库中，仅同步系统代码。

**手动备份**：建议定期将 `poems/` 文件夹手动备份到云盘或移动硬盘中。
